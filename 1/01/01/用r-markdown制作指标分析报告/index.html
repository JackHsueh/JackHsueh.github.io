用R+Markdown制作指标分析报告

1 Jan, 0001 · by Xuejianguo · Read in about 5 min · (2238 words) · Share this on:

R语言是一门免费且功能强大的数据分析编程语言，Markdown是一种简单但应用广泛的文本格式标记语言，两者结合便是报告制作利器RMarkdown. 网上介绍使用RMarkdown生成数据报告的总结资料很多，但实际学习起来还是难免遇到新的问题，特此自己也总结一下，以供以后自己或朋友们参考。

基本需求

原始数据为每月不同区域内不同生产单位的多台设备运行数据，需要生成不同区域的当月指标完成情况，主要包括： * 此区域指标目标值及本月实际完成值 * 此区域内各单位指标实际完成情况 * 各指标下未达成目标值设备列表 * 未达目标设备原始运行记录表 * 未达成原因分析及改进措施

由于最后一项需要各区域人员自行分析撰写，倒数第二项需要在分析后删除，所以输出文件需要为word格式

基本流程

报告制作流程

在UI界面中选择当月设备指标表和运行记录表；根据区域列表循环赋参运行Render函数；被调用的.Rmd文件接收参数，进行数据处理，并和文字生成.md文件；后台自动调用pandoc将.md文件转化为.docx文件

Rmd文件生成流程

在RStudio菜单中，点击新建R Markdown文件; 在弹出界面选择输出格式为Word，即可打开新建的.Rmd文件进行编辑；命名保存后，在RStudio菜单中，点击Knit to Word，即可生成Word文件；利用office word 软件打开生成的word文件，可编辑样式格式、页眉页脚等；再次新建.Rmd文件后，可在头文件中定义此文件为参考样式文件，则新生成的Word文件可得到相同的样式格式、页眉页脚1。

Rmd文件基本结构

.Rmd一般由三部分组成，文件头（YAML Header)、文本（Markdown Text)和代码块（Code Chunk), 每一部分需要遵循不同的语法。

文件头

定义文档元数据（如标题、作者、时间等），输出格式参数（output，如文档类型、参考样式文件、是否包含目录等）和变量参数（params，用于外部调用时传递参数）等。详细设置可参考RMarkdown手册2或Pandoc手册3。

文本

用于输入报告文本，可利用Markdown语法进行标记，可输入Latex语法表达的符号和公式，可使用用R语言表达式动态输入可变量，也可输入后续转化格式认可的语法（如OpenXML、HTML、LATEX，仅在相应格式下有效）。详见RMarkdown手册2

代码块

代码块内R代码可进行数据运算、输出字符、图表等。其用asis方式输出结果，也可被文本部分语法解析。详见Knitr手册4

文档内容实现

标题

Markdown中可以用headings 格式定义无编号标题：

# A level-one heading 
##  A level-two heading

可通过修改参考文件相应样式下的字体（字号、颜色）、段落（间距、段前分页、编号）等，实现所需的呈现效果。

正文

普通文本正常书写即可，其样式也可在参考文件中修改。局部强调、突出等可用Markdown格式修饰：

*Empahasis*
**Strong**
~~Strikethrough~~

对于引用的变量数据可以通过inline R code格式:`r command`在段落中替换输出：

Today is `r Sys.Date()`.

公式

行间公式用$...$将公共括起来，块间公式用$$...$$括起来，中间可用Latex语法书写公式:

The value of $h(\theta)$ can be calculated
$$h(\theta) = \sum_{j = 0} ^n \theta_j x_j$$

The value of $h(\theta)$ can be calculated:

$$h(\theta) = \sum_{j = 0} ^n \theta_j x_j$$

格式文本

如果输入的正文文本符合后续转化格式的语法，则可进一步得到解析。如输入的HTML语句

<p style="font-family:arial;color:red;font-size:30px;">HTML: red.</p>

在输出为html格式时，将按其语法解析为

HTML: red.

在输出为其它格式时一般无法得到相应效果，如在wordw仅输出正文文字，字体、颜色、字号等无效。

相应的，如果输入LATEX命令文本，

\LaTeX
\newpage

则在输出为pdf格式时，将其解析为$\LaTeX$显示和分页控制效果。

同样道理，在输入openxml文本时

​```{=openxml}
<w:p>
    </w:pPr>
    <w:r>
      <w:br w:type="page"/>
    </w:r>
  </w:p>
​```

输出输出转化为docx格式，则在相应位置产生分页效果。但相比上述html和pdf命令，繁琐且难掌握，一般不用。

绘图

图表输出可在代码块中用plot或ggplot函数输出，输出的参数，可在chunk option里面定义。

也可用RMarkdown语法插入图片链接，或用R 代码knitr::include_graphics导入后再输出。

表格

表格输出简单的方法是用knitr::kable()生成表格。因为本次需求所需表格样式比较复杂，如多表头、合并单元格、未达标数字标红等，因此选用了flextable包来生成表格，但其在循环输出时存在格式不符的问题，需要将结果收集起来，再统一输出5。

​```{r}
data(cars)
speed <- unique(cars$speed)
results <- character()
for (v in 1:length(speed)) {
  carspd <- cars[which(cars$speed == speed[v]),]
  tb <- regulartable(carspd)

  # Generate a figure in a temporary file
  filename <- tempfile(fileext = ".png")
  png(filename)
  hist(carspd$dist)
  dev.off()

  # Put everything into the results vector
  results <- c(results, "\n\nThis is the table for v =", v,
                        knit_print(tb),
                        knitr:::wrap(include_graphics(filename)))

}
asis_output(results)
knit_print(tb)
​```
